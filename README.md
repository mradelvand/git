## What You'll Learn

- How to safely publish personal projects without exposing private data
- The two-repository pattern: one public (code), one private (data)
- Core Git commands: `git rm --cached`, `git remote add`, `git cherry-pick`
- Shell aliases for a frictionless daily workflow
- Syncing data between machines using a private repo as the sync point
